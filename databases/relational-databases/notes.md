# Relational Databases

## Knowledge dump

I have worked with postgreSQL and MySQL. So, I'm familiar with relational databases. In every databases there are a lot of tables and each tables consists of multiple rows which we call as records across multiple columns which we call as fields. <br>
When we consider a real life entity such as an employee; an employee can have a lot of attributes such as name, salary, years of experience, department, project, days worked till now ,etc. And when you look into the department or project, we can have a lot of detials inside them. Where it is better to make themselves into a separate table. If we are making them into a separate table we have to connect employee and department/project somehow. And that is done using relationships between these tables. <br>
Relational databases model real-world entities as tables and connect them using relationships (typically through primary and foreign keys). This allows us to efficiently write queries to get the necessary data.
